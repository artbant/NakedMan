# NUKED MAN — Документ проекта

## Концепция
2D пиксельный платформер. Голый мужик в шляпе (15×24px) проснулся после вечеринки в другом измерении. Абсурдный юмор, враги говорят тарабарщиной, другой мир. Стиль: чистый ЧБ (#fff/#000), всё через `fillRect`, никаких изображений.

**GitHub:** artbant/NakedMan  
**Live:** https://artbant.github.io/NakedMan  
**Запуск локально:** `python3 -m http.server 8000` в папке проекта

---

## Архитектура файлов

```
nuked-man/
├── index.html     — canvas 100vw×56.25vw (16:9), мобильные кнопки
├── constants.js   — GW=480 GH=270, T=24, MH=11, rnd(), rndTex()
├── particles.js   — система частиц (sq, star, flash)

├── player.js      — спрайты, анимации, физика, удар, лестницы
├── enemies.js     — patrol/guard/runner, ragdoll, речевые пузыри, динамика Broforce
├── items.js       — бочки/взрывные бочки, пауэрапы (бутылка/сигарета)
├── hostages.js    — заложники с гравитацией, SAVE пузырь

```

**Порядок подключения скриптов важен:**
constants → particles → map → player → enemies → items → hostages → hud → game

- **↓ / S** — спуск по лестнице
- **Z / X** — удар
- **ESC** — пауза

## Текущее состояние

### Карта (map.js)
- Процедурная генерация: монолит, профиль высоты с перепадами, пещеры, пропасти, выступы
- Ширина уровня: 65+lvl×4 тайлов (прогрессия)
- Тайлы: чисто чёрный монолит (#000), белые трещины при повреждении
- Разрушаемость: хрупкий (1 удар), средний (2 удара), прочный (0)
- Лестницы: логичное размещение — только где перепад ≥3 тайла
- Портал в конце уровня
- tileHP, tileFlash — система разрушения с миганием

### Фон — 5 планов глубины
- **Небо:** градиент #111 (верх) → #777 (горизонт)
- **План 5:** #666, параллакс 0.02 — дальние холмы
- **План 4:** #444, параллакс 0.06
- **План 3:** #2a2a2a, параллакс 0.12
- **План 2:** #111, параллакс 0.22
- **Земля (тайлы):** #000 — максимальный контраст
- Каждый план — процедурный органичный силуэт через сумму синусов
- Снег: 90 частиц в 3 слоях с параллаксом и дрейфом

### Игрок (player.js)
- Спрайты STAY/RUN_0/RUN_1/PUNCH_0/1/2/JUMP через fillRect, цвет #fff
- Физика лестниц: ↑ подъём, ↓ спуск, гравитация отключена
- Invincibility frames 0.6 сек с миганием
- punchBreakTiles() — 3 тайла по вертикали перед кулаком

### Враги (enemies.js)
- **patrol** — силуэт #000, белые глаза #fff, боббинг при ходьбе
- **guard** — алертуется при приближении, стреляет, пузырь ? до алерта / ! при алерте, вспышка выстрела
- **runner** — появляется с уровня 4, убегает, мерцает, будит соседей
- Динамика Broforce: panicEnemiesNear() при взрыве, alertNearbyEnemies() при алерте guard
- Freeze frame 60-80мс при убийстве
- Ragdoll при смерти
- Речевые пузыри: тарабарщина над врагами (глитч-пиксели)

### HUD

## Ключевые технические решения
- Всё `fillRect` — никаких arc/font/drawImage
- Canvas: `width:100vw; height:56.25vw; max-height:100vh; max-width:177.78vh`

## СЛЕДУЮЩЕЕ ИЗМЕНЕНИЕ — Дизеринг фона

### Что хотим
Заменить серые цвета фоновых планов (#666, #444, #2a2a2a, #111) на **дизеринг** — паттерны чёрных точек на белом фоне разной плотности. Только два цвета: #000 и #fff. Референс: пиксельный дизеринг как в старых компьютерных играх и принтах.

### Как реализовать
Написать функцию `ditherFill(x, y, w, h, density)` через **упорядоченный дизеринг Bayer 4×4**:

```javascript
const BAYER4 = [
  [ 0,  8,  2, 10],
  [12,  4, 14,  6],
  [ 3, 11,  1,  9],
  [15,  7, 13,  5],
];

function ditherFill(x, y, w, h, density) {
  // density: 0.0 = белый, 1.0 = чёрный
  // threshold для каждого пикселя = BAYER4[py%4][px%4] / 16
  // если density > threshold — рисуем чёрный пиксель
  ctx.fillStyle = '#000';
  for (let dy = 0; dy < h; dy++) {
    for (let dx = 0; dx < w; dx++) {
      const threshold = BAYER4[(y + dy) % 4][(x + dx) % 4] / 16;
      if (density > threshold) {
        ctx.fillRect(x + dx, y + dy, 1, 1);
      }
    }
  }
}
```

### Применение к планам
- Небо (градиент): каждая строка рисуется ditherFill с density 0.05→0.45
- План 5: ditherFill, density 0.2
- План 4: ditherFill, density 0.35
- План 3: ditherFill, density 0.55
- План 2: ditherFill, density 0.75
- Земля (тайлы): остаётся #000 сплошным

### Важно
- Фон рисуется каждый кадр — попиксельный цикл дорогой
- Оптимизация: рисовать только видимую область (0..GW по X, 0..GH по Y)
- Силуэтные профили планов рисуем поверх уже нанесённого дизеринга
- Каждый план = сначала ditherFill прямоугольником от горизонта вниз, потом вырезаем профиль силуэта сверху

Только **map.js** — функции `drawBgFar()` и `drawBgMid()`
