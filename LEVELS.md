# LEVELS.md — Создание уровней

Этот гайд объясняет как рисовать уровни руками в файле `levels.js`.

## Общая идея

Каждый уровень — массив строк. Каждый символ в строке = один тайл (24×24 пикселя в мире). Высота уровня всегда **11 строк** (константа `MH`). Ширина — любая, но рекомендуется от 100 до 200 тайлов.

```js
const LEVELS = [
  // уровень 1
  [
    "                                      ",
    "                                      ",
    "                                      ",
    "                  F                   ",
    "                                      ",
    "      E           G                   ",
    "                ######                ",
    "                                      ",
    "@..........................P.........",
    "######################################",
    "######################################",
  ],
  // уровень 2 ...
];
```

Если `LEVELS[lvl - 1]` существует — игра загружает ручной уровень. Если нет — запускается процедурная генерация.

## Легенда символов

### Геометрия (тайлы)

| Символ | Значение |
|--------|----------|
| `#` | Монолит обычной прочности. На поверхности (глубина 0-1) = 1 HP, в середине (2-4) = 2 HP, глубокие (5+) = 3 HP |
| `X` | Хрупкий монолит (**всегда 1 HP** — ломается с одного удара) |
| `O` | Крепкий монолит (**всегда 3 HP** — нужно 3 удара) |
| пробел или `.` | Воздух (ничего) |

### Спаун-объекты

| Символ | Значение |
|--------|----------|
| `@` | Позиция старта игрока. Если не указан — ставится на x=2 |
| `P` | Портал (финиш уровня). Если не указан — ставится автоматически в конце карты |

### Враги

| Символ | Тип | Размер | HP |
|--------|------|--------|-----|
| `E` | patrol (ходит туда-сюда) | 18×24 | 2 |
| `G` | guard (стреляет) | 16×22 | 1 |
| `R` | runner (убегает) | 16×22 | 1 |
| `F` | flyer (летает) | 20×14 | 1 |
| `S` | sniper (снайпер) | 14×22 | 2 |
| `U` | bruiser (громила) | 24×30 | 3 |
| `B` | босс (Глаз) | 48×48 | зависит от уровня |

### Предметы

| Символ | Значение |
|--------|----------|
| `b` | Бочка обычная (можно пнуть, ломается) |
| `x` | Взрывная бочка (при ударе — взрыв с радиусом) |
| `m` | Аптечка (+1 HP, до максимума 3) |
| `i` | Сигарета (speed boost 5 сек) |
| `H` | Заложник (можно спасти — очки + музыкальная фанфара) |

## Правила

1. **Высота всегда 11 строк.** Если меньше — дополняются пустые. Если больше — обрезается до 11.
2. **Ширина нормализуется автоматически** — короткие строки дополняются пробелами справа до максимальной длины.
3. **Один тайл = 24×24 px в мире.** Если хочешь 2-тайловую стену, пиши два `#` подряд.
4. **Враги/предметы размещаются на указанной строке, но автоматически "оседают" на землю** — парсер ищет ближайший твёрдый тайл снизу и ставит туда.
5. **Границы карты не закрываются автоматически** — если сделаешь уровень без пола, игрок упадёт в бесконечность. Ставь нижние строки `#########...` как пол.
6. **Пробелы и точки равнозначны** — используй пробелы для "пустоты" или точки для "открытого пространства где может быть что-то". Это чисто вопрос читаемости.

## Типовые шаблоны

### Простая плоская земля

```
                                   
                                   
                                   
                                   
                                   
                                   
                                   
                                   
@............................P....
###################################
###################################
```

### Плоскость с препятствием

```
                                   
                                   
                                   
                                   
         X                         
       ####                        
                                   
                                   
@........................P........
###################################
###################################
```

Символ `X` — хрупкий блок в воздухе, можно разбить чтоб упасть, либо пройти снизу.

### Пропасть

```
                                   
                                   
                                   
                                   
                                   
                                   
                                   
                                   
@..............         ..........P
##############          ###########
##############          ###########
```

Пропасть шириной 9 тайлов. Требует прыжка, можно + double jump.

### Многоуровневый рельеф

```
                                            
                                            
                                            
                                            
                       #########            
                  ######                    
           #######                          
     ######                                 
####                                      ##
@......                                  ..P
#######################################   ##
```

Подъём лестницей. Хорошо работает в сочетании с `F` (летающими врагами сверху).

### Засада снайпера

```
                                                
                                                
                                                
           S                                    
           ##                                   
           ##                                   
                                                
                                                
@................                         .....P
##################      #######################
##################      #######################
```

Снайпер стоит на возвышении у пропасти. Пока пытаешься прыгнуть — стреляет. Даёт момент напряжения.

### Арена-мясорубка

```
                                                         
                                                         
      F                F                     F          
                                                         
                                                         
   E         G         R          S           U          
                                                        B
@........................................................
##########################################################
##########################################################
```

Все 6 видов врагов + босс в конце. Хорошо для теста анимаций и поведения.

### Запертая комната с боссом

```
                                            
                                            
                                            
                                            
                                           B
                                            
#####           ##############              
    #           #                           
    #           #                           
@...#...........#...........P               
####################################        
```

Чтобы добраться до босса, надо пройти лабиринт из стенок.

## Последовательность действий при создании уровня

1. **Определи идею** — чем этот уровень интересен? (Пропасть? Арена? Засада? Лабиринт?)
2. **Нарисуй пол** — нижние 1-2 строки `############`
3. **Поставь старт `@`** — обычно на x=0 или 1, на верхнем слое пола
4. **Поставь портал `P`** — обычно в самом конце карты
5. **Добавь препятствия** — блоки, возвышения, пропасти
6. **Расставь врагов** — начинай с простых (patrol, guard), потом сложные
7. **Добавь бонусы** — 1-2 аптечки `m`, 1 сигарета `i` если сложно
8. **Босс** `B` — можно поставить в конкретную позицию. Если не указать, босс всё равно спавнится автоматически в стандартной позиции.

## Часто задаваемые вопросы

### "Игра падает при запуске уровня"

Скорее всего строка `@` попала в воздух (нет твёрдого тайла снизу). Убедись что под `@` есть `#`.

### "Враг не появляется"

- Проверь что символ правильный (`E` не `e`)
- Проверь что под врагом есть земля
- Проверь что враг не перекрыт тайлом сверху

### "Босс не появляется"

Босс появляется на **каждом** уровне, даже если не указан `B` в разметке. Если указан `B` — он встанет в этой позиции, иначе в стандартной (конец карты).

### "Портал не работает"

Убедись что на этом уровне нет живого босса. Портал блокируется пока босс жив.

### "Можно ли делать вертикальные шахты?"

Да, строки идут сверху вниз. Создай "туннель" из воздушных пробелов через пол:
```
#####     #####
#####     #####
#####     #####
########### ###
```

### "Что будет на уровнях после последнего ASCII?"

Если ASCII уровней, например, 3, то на уровне 4 и дальше игра переходит к **процедурной генерации**. Так что можно сделать первые 3-5 "ручных" тутор-уровней, потом отдать процедуре.

## Советы

- **Колонки легко считать** по индексу символа в строке (0, 1, 2, 3...). Удобно использовать редактор с линейкой.
- **Используй моноширинный шрифт** в редакторе (VS Code, Sublime) — тогда все символы одинаковой ширины и карта читается визуально.
- **Не делай слишком плотно** врагов — между ними нужно ~5-10 тайлов дыхания. Иначе игрок моментально получит урон и не увидит разнообразия.
- **Ставь аптечку перед сложным сегментом** — это создаёт ритм "испытание → награда → испытание".
- **Начинай с простого** — первые уровни должны быть легче. Процедурная генерация автоматически усложняет с ростом `lvl`, но для ручных уровней это нужно делать самой.
- **Сохраняй и тестируй сразу** — открой игру, нажми F5 (с Disable cache), пройди уровень. Правь если нужно.

## Пример полного уровня с комментариями

```js
// ───────── УРОВЕНЬ 2 — "Башня снайпера" ────────────────────────
// Игрок начинает внизу. Слева — patrol, играет вводную.
// В середине — обрыв с бочкой-взрывчаткой для расчистки пути.
// Справа — поднимающаяся башня со снайпером наверху.
[
  "                                                        ",
  "                                                        ",
  "                                        S               ",
  "                                        ##              ",
  "                                        ##              ",
  "                                       ###              ",
  "                    x                  ###              ",
  "              m     #                 ####              ",
  "@........E........###..........#######################P ",
  "#########################                ###############",
  "#########################                ###############",
],
```
